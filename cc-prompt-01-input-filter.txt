GOAL: Add an input filter that strips junk rows BEFORE they enter the classification pipeline. This filter lives in the shared pipeline so both CLI and API benefit. Junk rows are inventory labels, build configs, and internal names that are not real hardware model numbers. They have no manufacturer and no recognizable part number pattern.

STEP 1 — Create src/eol_tool/input_filter.py

Create a module with these functions:

def is_junk_row(model: str, manufacturer: str) -> bool
    Returns True if the row should be filtered out. A row is junk if ALL of these are true:
    - manufacturer is empty or blank
    - model does NOT match any known hardware pattern

    Known hardware patterns that should NOT be filtered (even without manufacturer):
    - Optics: contains SFP, QSFP, XFP, CFP, or matches transceiver naming like 10G, 40G, 100G
    - Part numbers: contains a dash with alphanumeric segments on both sides (like ASR-6405E)
    - Drive model numbers: starts with a digit and contains letters (like 0F12470)

    Junk patterns that SHOULD be filtered:
    - Pure size/capacity descriptions: matches patterns like "1U 10BAY", "2U 12BAY"
    - Server build configs: contains "SERVER BAREBONE", "SERVER:USED", "RW SERVER"
    - Internal inventory codes: "UK-4210", "UK-4216", "AMS1 SERVER"
    - Vague labels: "NEW", "USED", "FS BOX", "HALF-SLIM SSD"
    - RAM config strings: matches patterns like "2 X 4316 - 768GB", "4216 - 128GB - 480GB"
    - CPU config strings that are not actual CPU models: "E3-1200 SERIES" (the word SERIES)
    - Short cryptic codes with no manufacturer: "4CH 21-24", "PTTS-H008-0-02M", "SCLA15TQC-563W", "1TBI 7K", "7960 1TBRAM"

def filter_models(models: list) -> tuple:
    Takes a list of HardwareModel objects.
    Returns a tuple of (clean_models, filtered_models).
    filtered_models is a list of dicts with model, manufacturer, and reason for filtering.
    Log each filtered row with the reason.

STEP 2 — Integrate into the pipeline

Find where models are loaded and passed to checkers — this is the shared pipeline path used by both CLI and API. Insert the filter AFTER reading and normalizing, BEFORE checking. The filter should:
