Read src/eol_tool/checkers/tech_generation.py and src/eol_tool/checkers/manual.py and src/eol_tool/models.py to understand the current classification system.

GOAL: Generic white-label optics (SFP, QSFP, XFP, CFP transceivers) with no manufacturer should be classified as ACTIVE with risk_category NONE. These are commodity items — interchangeable, always available from multiple vendors, and never go EOL in any meaningful sense. They should NOT remain as UNKNOWN.

STEP 1 — Create src/eol_tool/checkers/generic_optics.py

A new checker that handles optics with no manufacturer (or manufacturer set to a generic/unknown value).

class GenericOpticsChecker with BaseChecker:
    manufacturer_name = "__generic_optics__"

This checker should:
- Activate when manufacturer is empty, blank, "unknown", or "generic"
- Match models that look like optics transceivers using these patterns:
  - Contains SFP, QSFP, QSFPP, XFP, or CFP anywhere in the model string
  - Contains transceiver speed indicators: 1G, 10G, 25G, 40G, 50G, 100G, 400G
  - Contains fiber/copper indicators: SR, LR, ER, ZR, SR4, LR4, PLR4, UNIV, BASE-T, COPPER
  - Contains DWDM channel indicators: DW followed by digits, or C followed by 1-2 digits then space
  - Contains converter indicators: CVR-QSFP
- Return status ACTIVE with:
  - eol_reason: none (commodity items dont have EOL)
  - risk_category: NONE
  - confidence: 80
  - source_name: "generic-optics-classifier"
  - notes: "commodity-transceiver-always-available" for standard optics
  - notes: "dwdm-optic-commodity" for DWDM channel optics

For models that look like optics but WITH a known manufacturer (like Juniper JNP-QSFP or HPE branded optics), return NOT_FOUND so the vendor-specific checker handles them.
